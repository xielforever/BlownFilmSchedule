# 吹膜排程 MVP 技术架构

## 架构

- `frontend/`：Vite + React + TypeScript 单页排程工作台。
- `backend/`：FastAPI 服务、Excel 解析、排程算法和导出。
- `data/machines.xlsx`：本地机台能力表，作为 MVP 的机台配置来源。
- `examples/`：基于截图订单整理的生产压力样例工作簿。
- `outputs/`：排程结果、约束审计和 Markdown 报告。

## 核心数据流

1. 前端启动时通过 `GET /api/machines` 读取 `data/machines.xlsx` 并展示机台摘要。
2. 前端上传订单 Excel 到 `POST /api/schedule/preview`。
3. 后端读取 `orders` sheet，再加载 `data/machines.xlsx` 中的本地机台能力。
4. 后端完成输入质量校验、规格解析和候选机台约束审计。
5. 前端展示校验结果、机台摘要和订单统计。
6. 用户触发 `POST /api/schedule/run`。
7. 后端执行排程算法并写出导出文件。
8. 前端展示甘特图、任务审计、异常表、约束表，并提供下载。

## 排程算法

MVP 采用确定性启发式：

- `orders` 表只表达订单需求，不承载机台、开始时间、结束时间。
- 机台能力由 `data/machines.xlsx` 提供，订单上传不承载机台配置。
- 解析规格得到宽度、厚度、插边宽。
- 算法遍历所有本地机台自动选机。
- 校验机台产能、宽度区间、插边能力、`HD_ONLY`、`SF101_ONLY`、`NON_BLOW_RATIO_RULE`。
- 机台适配分为 `best`、`recommended`、`marginal`、`blocked`；边界可做机台会进入候选但带评分惩罚。
- 对每个订单遍历候选机台和可插入窗口。
- 生产时长优先按订单批量和候选机台平均产能估算。
- 换料、调宽、调厚、调插边会折算为换型时长，计入任务总占用。
- 使用评分选择最优插入：
  - 延期惩罚最高。
  - 其次惩罚换料、调机、换型时长和空档浪费。
- 所有决策写入审计说明。
- 每个订单保留主要候选机台审计，包含候选排名、评分差值、可开工/完成时间、换型时长和未选原因。

## 订单输入质量

预览阶段只校验订单输入本身，不接收人工计划或外部排程状态：

- 必填列、规格解析、交期、批量等基础字段。
- 重复订单号、交期早于订单日期、缺交期、缺有效批量。
- 订单日期到交期间隔过长，避免 mock 或导入数据过松导致排程解释失真。
- 同交期同配方订单集中到期，提示后续可能被算法分散到多台机。
- `order_qty * unit_weight_g` 与 `batch_kg` 偏差过大。
- `batch_kg / work_hours` 折算产能明显异常。

## 完整字段预留

MVP 不使用但已在模型或样例工作簿中预留：

- 客户、洁净等级、是否医用、颜色、是否允许拆单。
- 损耗率、最小开机量、清洁验证、批次隔离。
- 换配方、换模、换宽、换厚、换插边的真实时间规则。
